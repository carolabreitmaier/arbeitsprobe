# arbeitsprobe
Der Code ist in einem Jupyter-Notebook gespeichert. Durch ein Klick auf "Cell" --> "Run All" kann der gesamte Code für die Schritte 1 und 2 der Arbeitsprobe ausgeführt werden.

1) Zuerst verschaffte ich mir einen Überblick über die .csv-Datei und das NumPy-Array, um ein Verständnis über die Struktur und die Daten zu erhalten. Für die folgenden Schritte zog ich ChatGPT als Unterstüzung heran. Mit Hilfe von ChatGPT berechnete ich für jede:n der 50 Proband:innen drei graphentheoretische Kennzahlen, welche ich in einer neuen Matrix mit dem Namen "features" speicherte. Bei Betrachtung der ersten drei Einträge (die Werte der ersten drei Proband:innen) zeigte sich eine Auffälligkeit - die Matrix enthielt komplexe Zahlen. Dies hatte zur Folge, dass mit dieser Matrix kein Klassifikationsmodell gebaut werden konnte. Ich entschied mich, die negativen Korrelationswerte in der ursprünglichen Konnektivitätsmatrix zu entfernen bzw. deren Beträge zu nehmen. Hierbei stellt sich jedoch die Frage, ob das theoretisch machbar ist. Der ursprüngliche Datensatz wird verfälscht und Daten werden entfernt, was sich letzendlich auf das Klassifikationsmodell auswirkt. Der Umgang mit den komplexen Zahlen müsste vor den kommenden Schritten geklärt werden, denn so kommt es zu Folgefehlern und einem fehlerhaften Modell. Eine weitere Schwierigkeit, die sich bei der Erstellung eines Klassifikationsmodells ergibt ist, dass nicht eindeutig ist, ob die Patient:innen oder die Kontrollpersonen die Gruppe 1 oder 2 bilden (bezogen auf DX_GROUP in der .csv-Datei). So ist eine Klassifizierung, ob eine Person an Autismus erkrankt ist oder nicht, nicht möglich. Zudem zeigen die Ergebnisse der fünfachen Corss-Validation, dass das Modell nicht vorhersagekräftig ist - höchstwahrscheinlich eine Folge am Umgang mit den komplexen Zahlen. 

2) Wenn ich für diesen ABIDE-Datensatz genau drei Monate Zeit hätte und ein Paper über "Dynamische Netzwerke bei ASD" schreiben würde, dann würde ich erstens Methoden heranziehen, die Störvariablen wie beispielsweise das Alter kontrollieren. Bei fMRI ist eine besondere Herausforderung, dass Bewegungen der Proband:innen im Scanner zu Artefakten und verfälschten fMRI-Daten führen. Diese Bewegungsartefakte führen zu verfälschten Korrelationen in der Konnektivitätsmatrix, die widerum zu verfälschten graphentheoretischen Metriken führen. Gleichzeitig kann es dazu kommen, dass ML-Modelle nicht die Krankheit klassifizieren, sondern Bewegungsmuster im Scanner. Ich würde eine Methode heranziehen, die den Einfluss von Bewegungsartefakten reduziert und somit die Daten und Kennwerte um diesen Einfluss bereinigt. Beispielsweise gibt es Methoden, die besonders bewegungsintensive Zeitpunkte (definiert über einen Threshold) identifizieren und aus der Analyse ausschließen (z.B. framewise displacement thresholding). Ich würde mich in meinem Story-Telling auf die Kontrolle von verschiedenen Störvariablen (z.B. demographische Daten wie Alter und Bewegungsartefakte) fokussieren und die Daten des ABIDE-Datensatzes bereinigen, sodass die Klassifizierung, ob eine Person an Autismus erkrankt ist oder nicht, aussagekräftiger ist.
